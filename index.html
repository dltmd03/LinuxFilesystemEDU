터미널로 바로 배우기

경로(/, ~, ..), 디렉터리/파일 구조, 핵심 명령어를 인터랙티브 터미널로 실습하세요. 내부 구현은 트리 + 이중연결리스트를 사용합니다.

실습 시작 개념 먼저 보기
Filesystem project — 10114 이승진
터미널 미리보기
$ cd ~
$ mkdir -p school/2025/report
$ touch ~/school/2025/report/readme.txt 120
$ tree ~/school
└─ school/
  └─ 2025/
    └─ report/
      └─ readme.txt (120)
📁

파일시스템이란?

쉽게 말해서 “저장장치 위에 파일과 폴더를 정리하는 방법” 입니다. 우리 집 책장에 책을 꽂듯이, 컴퓨터도 파일을 어디에 어떻게 둘지 정해두어야 합니다.

  • 파일이 어디에 있는지 기록하는 지도 역할
  • 폴더 안에 폴더가 계속 들어가는 트리(Tree) 구조
  • 삭제·복사·이동이 안전하게 동작하도록 도와줌
🌐

어디에서 쓰이나요?

우리가 직접 보지 못할 뿐, 리눅스 파일시스템은 엄청 많은 곳에서 쓰입니다.

  • 웹 서버, 게임 서버, DB 서버 같은 대부분의 서버
  • AWS·GCP·Azure 같은 클라우드 환경의 리눅스 서버
  • 안드로이드, 라즈베리파이 등 임베디드/IoT 장비
  • 보안·해킹·포렌식에서 다루는 리눅스 시스템 분석
🎯

이 사이트에서 배우는 것

단순히 명령어만 외우는 게 아니라, “안에서 어떻게 돌아가는지”까지 보는 것이 목표입니다.

  • /, ~, .. 같은 경로 개념과 디렉터리 구조
  • 트리 + 이중연결리스트로 파일시스템을 자료구조로 표현하는 법
  • mkdir, rm -r, cp 등이 내부에서 하는 일
  • 개발·보안 실무에서 꼭 필요한 기본 터미널 실력

누가 보면 좋은가요?

  • 윈도우만 써봤는데 리눅스가 궁금한 사람
  • 백엔드·서버·클라우드·보안 쪽에 관심 있는 학생
  • 리눅스 명령어는 조금 쳐봤지만, 구조는 잘 모르는 사람
  • 트리/리스트 같은 자료구조를 실제 예제로 느껴보고 싶은 사람

추천 학습 흐름

  1. 이 홈 화면에서 파일시스템이 무엇인지 감 잡기
  2. 개념 / 명령어 섹션에서 필수 이론 훑기
  3. 실습 터미널에서 직접 폴더 만들고 지워보기
  4. 실습 챌린지(quiz)로 원하는 구조까지 만들어보기
  5. 막히는 부분은 커뮤니티에 터미널 출력 붙여넣어 질문

리눅스, 처음 보는 사람을 위한 3분 설명

리눅스는 한마디로 말하면 “서버와 개발자가 정말 많이 쓰는 운영체제” 입니다. 겉모습은 윈도우와 다르게 보일 수 있지만, 결국 하는 일은 비슷합니다.

다만 화면을 클릭하는 대신, 터미널에서 명령어를 치는 방식에 더 초점이 맞춰져 있어서 처음에는 낯설 수 있지만, 몇 가지 패턴만 익히면 금방 편해집니다.

1) 리눅스는 어디에 있을까?

  • 우리가 접속하는 웹/게임/채팅 서버
  • 클라우드(AWS, GCP, Azure)의 대부분 서버
  • 안드로이드, 라즈베리파이 같은 임베디드 기기

2) 왜 개발자/보안 쪽에서 많이 쓸까?

  • 무료이고 커스터마이징이 자유로움
  • 서버 관리, 자동화 스크립트 작성에 최적화
  • 보안/포렌식 도구들이 대부분 리눅스 기준

3) 이 사이트에서 어디까지 알면 될까?

  • 리눅스가 파일/폴더를 어떻게 생각하는지
  • 경로 개념과 기본 명령어
  • 실제 내부 구현을 트리 + 리스트로 보는 감각

터미널이 무섭게 느껴지는 이유

  • 화면이 전부 검은색 + 글자뿐이라 낯설다
  • 명령어 하나 잘못 치면 다 날아갈 것 같은 느낌
  • 어디에 있는지(현재 위치) 감이 안 잡힌다

이렇게 생각하면 훨씬 편해져요

  • 터미널은 사실 “채팅창 같은 입력창”일 뿐이다.
  • 명령어는 “짧은 문장”이라서 금방 외운다.
  • 이 사이트에선 실제 파일이 아니라 가상 파일이라 마음 편하게 실수해볼 수 있다.

이 페이지에서 먼저 익숙해지면 좋은 키워드

/ (루트 디렉터리) ~ (내 홈 디렉터리) .. (부모 디렉터리) cd (이동) ls (목록 보기) mkdir (폴더 만들기) rm -r (폴더 지우기)

아래의 개념 섹션에서는 파일시스템 구조를, 명령어 섹션에서는 실제로 많이 쓰는 터미널 명령을, 실습 섹션에서는 이 모든 것을 직접 손으로 쳐 보면서 익히게 됩니다.

핵심 개념: 리눅스 파일시스템 한 번에 정리

큰 그림
모든 것은 / 아래

리눅스 파일과 폴더는 전부 루트 디렉터리(/)에서 시작하는 하나의 큰 트리 구조로 이어져 있습니다.

길 찾기
경로와 현재 위치

절대 경로 / 상대 경로, ., .., ~ 만 확실히 알면 웬만한 길 찾기는 다 해결됩니다.

실습과 연결
트리 + 이중연결리스트

이 사이트의 실습 터미널은 실제 파일 대신, 트리 + 이중연결리스트로 만든 가상 파일시스템을 조작합니다.

1. 파일시스템이란?

한 줄로 말하면, “저장장치 위에 파일과 폴더를 정리하는 규칙”입니다. 리눅스에서는 이 정리가 나무(Tree)처럼 생겼고, 그 꼭대기가 / 입니다.

  • 윈도우의 C:\, D:\ 대신, 큰 나무 하나(/)가 있다고 생각하면 됩니다.
  • USB·외장하드 같은 추가 저장장치도 이 트리에 “붙여서(mount)” 사용합니다.
  • 이 트리 구조에 대한 대략적인 규칙을 FHS(Filesystem Hierarchy Standard)라는 표준이 정합니다.

2. 경로(path)와 현재 작업 디렉터리

파일을 가리킬 때는 항상 경로(path)를 사용합니다. 경로는 폴더 이름을 / 로 이어 붙인 문자열입니다.

절대 경로 (absolute)

/ 로 시작, 항상 같은 위치

  • /usr/bin/ls, /home/user/docs
  • 어디에서 실행하든 동일한 파일을 가리킵니다.
상대 경로 (relative)

pwd 기준, 현재 위치에 따라 달라짐

  • 현재가 /home/user 일 때,
    docs/report.txt → /home/user/docs/report.txt
  • 터미널에서 작업할 때 가장 많이 쓰이는 형태입니다.

자주 쓰는 특수 기호

. : 현재 디렉터리
예) ./run.sh
.. : 부모 디렉터리
예) cd ..
~ : 내 홈 디렉터리
예) cd ~/projects 
$ pwd
/home/user
$ cd ..
$ pwd
/home
$ cd ~/projects
$ pwd
/home/user/projects

3. 리눅스 상위 디렉터리 지도

아래 디렉터리들만 알아도 “대충 어디에 뭐가 있을지” 감이 잡힙니다.

/ 루트

모든 디렉터리의 시작점. /home, /etc, /usr 등이 여기 아래에 있습니다.

/home 사용자 공간

일반 계정의 개인 폴더. 예: /home/alice. 문서·사진·코드 등 대부분의 작업 파일이 여기에 있습니다.

/root 루트 홈

관리자 계정(root)의 홈 디렉터리. 일반 사용자 홈과 분리되어 있습니다.

/bin, /usr/bin

ls, cp, mv 같은 실행 파일들이 들어 있습니다. 우리가 치는 기본 명령어 대부분이 이쪽에 있습니다.

/etc 설정 파일

계정, 네트워크, 서버 등의 시스템 설정이 저장되는 곳입니다. 텍스트 기반 설정 파일이 많습니다.

/var, /tmp

/var : 로그(/var/log)와 자주 바뀌는 데이터.
/tmp : 임시 파일. 재부팅 시 사라질 수 있습니다.

/dev, /proc, /sys

장치와 시스템 정보를 파일처럼 다루게 해주는 특수 디렉터리입니다. 예를 들어 /proc/cpuinfo를 열면 CPU 정보가 텍스트처럼 보입니다.

파일시스템을 “나무(Tree)”로 본 그림

루트(/)가 뿌리(root)이고, 그 아래로 home, etc, usr 가지가 뻗어 나가며, 다시 그 아래에 사용자·프로그램 디렉터리들이 “잎”처럼 달려 있는 모습으로 보면 됩니다.

4. 윈도우랑 비교해서 감 잡기

📁 사용자 폴더

윈도우: C:\Users\이승진\Documents
리눅스: /home/이승진/Documents

💽 드라이브 / 디스크

윈도우: C:\, D:\ 로 나뉨
리눅스: 보통 /mnt 나 /media 아래에 디스크가 마운트됨

🧩 프로그램 설치

윈도우: C:\Program Files
리눅스: /usr, /opt 등 디렉터리로 나뉘어 설치

5. 이 사이트 실습과 자료구조 연결

① 트리(Tree)로 전체 디렉터리 표현

루트(/)가 최상단 노드이고, 그 아래에 home, etc 같은 자식 노드들이 붙습니다.

② 이중연결리스트로 형제 정렬

같은 디렉터리 안의 항목들은 이중연결리스트로 이어져 있어 이름순 정렬을 유지하면서 삽입/삭제를 빠르게 처리합니다.

③ 명령어 = 자료구조 조작 함수

cd 는 “현재 노드 이동”,
mkdir 는 “자식 노드 추가”,
rm -r 는 “서브트리 삭제”처럼
각 명령어가 트리/리스트 구조와 1:1로 대응되도록 구현되어 있습니다.

여기까지 이해했다면, 이제 “명령어” 섹션을 한 번 훑어보고 바로 위 메뉴의 실습 터미널에서 손으로 직접 쳐 보면서 머릿속 파일시스템 지도를 완성해 보세요 🙂

자주 쓰는 리눅스 명령어

이동 · 조회
  • pwd 현재 경로
  • cd [PATH|~] 디렉터리 이동
  • ls [-l] [-a] [PATH] 목록 (기본 가로 출력)
  • tree [PATH] 트리 출력
  • stat [PATH] 메타정보
생성 · 삭제 · 수정
  • mkdir [-p] PATH 디렉터리 생성 (새 폴더 파란색 표시)
  • rmdir PATH 빈 디렉터리 삭제
  • touch PATH [size] 파일 생성/크기 설정
  • rm [-r] [-f] PATH 파일/디렉터리 삭제
  • mv SRC DST 이동/이름변경
  • cp SRC DST 파일 복사

주의: 실제 Linux의 권한/하드링크/심볼릭링크/글로빙 등은 단순화되어 있습니다(교육용).

실습 터미널

별칭: ll=ls -l, la=ls -a, lsa=ls -la

학습 로드맵

  1. 1. 경로와 현재 위치 익히기

    먼저 “지금 내가 어디에 있는지”부터 익힙니다. 경로 개념이 잡히면 나머지는 훨씬 쉬워집니다.

    • pwd 로 현재 디렉터리 확인
    • cd, .., ~ 로 이리저리 움직여 보기
    • /home/user 처럼 절대 경로와 상대 경로 차이 느껴보기
  2. 2. 파일·폴더 직접 만들어 보기

    이제 손으로 만져보면서 감을 잡는 단계입니다. 폴더를 만들고, 파일을 만들고, 지우는 경험이 중요합니다.

    • mkdir, mkdir -p 로 폴더 구조 만들기
    • touch 로 파일 생성 + 크기 지정해 보기
    • rm, rm -r, rmdir 로 지우는 연습
  3. 3. 트리와 상태 확인하기

    내가 만든 구조가 어떻게 생겼는지 눈으로 확인해 봅니다. “지도 보는 법”을 배우는 단계입니다.

    • ls, ls -l, ls -a 출력 비교
    • tree 로 전체 구조 한 번에 보기
    • stat 으로 파일의 크기·시간 정보 확인
  4. 4. 나만의 작은 프로젝트 설계

    마지막으로 실제 쓸 만한 폴더 구조를 직접 설계해 봅니다. 리포트, 코딩 프로젝트, 블로그 등 아무거나 상상해도 좋습니다.

    • mkdir -p 로 school/2025/report 같은 계층 구조 만들기
    • 필요한 자리마다 touch 로 더미 파일 채워 넣기
    • tree 로 “내 프로젝트 폴더”가 마음에 드는지 확인

TIP: 한 번에 완벽하게 하려고 하지 말고, 각 단계를 5~10분 정도씩 가볍게 여러 번 반복해 보세요. 개념 섹션의 설명 → 로드맵 단계 → 실습 터미널을 왔다 갔다 하면서 익히면, 리눅스 초보자도 자연스럽게 파일시스템에 익숙해질 수 있습니다.

FAQ

실제 리눅스와 100% 동일한가요?

교육용으로 단순화했습니다. 권한, inode, 심볼릭링크 등은 생략돼 있습니다. 다만 경로 해석, 기본 명령 흐름은 리눅스와 유사합니다.

폴더가 파란색으로 보이는 이유는?

mkdir로 만든 디렉터리는 학습 가시화를 위해 파란색으로 강조됩니다(리눅스의 컬러 ls와 유사한 피드백).
